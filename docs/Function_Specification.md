# ClothCast 기능 명세서

## 1. 프로젝트 개요

**ClothCast**는 사용자가 등록한 옷장 데이터를 기반으로, 현재 날씨와 사용자의 일정/상황 프롬프트를 반영해 AI가 적절한 코디를 추천해주는 웹 서비스이다.

예시:

- 날씨: `17도`
- 프롬프트: `나 면접 가`
- 사용자 옷장: 블레이저, 셔츠, 슬랙스 보유

결과:

- 면접 상황과 날씨에 맞는 포멀 코디 추천

## 2. 기능 목록

### 2.1 회원가입

기능 설명:

사용자는 이메일과 비밀번호를 이용해 회원가입할 수 있다.

상세 조건:

- 이메일은 중복될 수 없다.
- 비밀번호는 최소 길이 제한이 있다.
- 회원가입 성공 시 로그인 페이지 또는 메인 페이지로 이동한다.

입력값:

- 이메일
- 비밀번호
- 닉네임

출력값:

- 회원가입 성공 여부
- 실패 시 에러 메시지

예외 처리:

- 이미 가입된 이메일인 경우
- 이메일 형식이 올바르지 않은 경우
- 비밀번호 조건을 만족하지 못하는 경우

### 2.2 로그인

기능 설명:

사용자는 가입한 계정으로 로그인할 수 있다.

상세 조건:

- 이메일/비밀번호가 일치해야 한다.
- 로그인 성공 시 사용자 세션이 유지된다.
- 로그인 상태에서만 옷장 관리 및 추천 기능을 사용할 수 있다.

입력값:

- 이메일
- 비밀번호

출력값:

- 로그인 성공 여부
- 사용자 기본 정보
- 인증 토큰 또는 세션 정보

예외 처리:

- 존재하지 않는 계정
- 비밀번호 불일치
- 인증 실패 시 에러 메시지 반환

### 2.3 로그아웃

기능 설명:

로그인된 사용자는 로그아웃할 수 있다.

상세 조건:

- 로그아웃 시 세션 또는 토큰이 제거된다.
- 로그아웃 후 보호 페이지 접근 시 로그인 페이지로 이동한다.

### 2.4 사용자 인증 상태 확인

기능 설명:

앱 진입 시 현재 로그인 상태를 확인한다.

상세 조건:

- 로그인 상태이면 사용자 정보를 유지한다.
- 비로그인 상태이면 공개 페이지만 접근 가능하다.

활용 목적:

- 헤더 UI 분기
- 보호 라우팅 처리
- 사용자별 데이터 요청

## 3. 옷장 관리 기능

### 3.1 옷 등록

기능 설명:

사용자는 자신의 옷을 등록할 수 있다.

입력 항목:

- 옷 이름
- 카테고리
- 색상
- 핏
- 포멀함 점수
- 소재
- 패턴
- 이미지 파일

카테고리 예시:

- 상의
- 하의
- 아우터
- 신발

핏 예시:

- 오버사이즈
- 레귤러
- 슬림

의류 데이터 구조:

- `id`: 의류 고유 ID
- `userId`: 의류를 등록한 사용자 ID
- `name`: 옷 이름
- `category`: `TOP`, `BOTTOM`, `OUTER`, `SHOES` 중 하나
- `color`: 색상
- `fit`: `oversized`, `regular`, `slim` 중 하나
- `formality`: 포멀함 점수, `1~5`
- `material`: 소재, 선택값
- `pattern`: 패턴, 선택값
- `imageUrl`: 업로드된 이미지 URL
- `createdAt`: 등록일

상세 조건:

- 필수값은 이름, 카테고리, 색상, 핏, 포멀함 점수, 이미지로 한다
- 소재와 패턴은 선택값으로 저장할 수 있다
- 포멀함 점수는 `1~5` 범위만 허용한다
- 이미지 업로드 성공 후 이미지 URL을 저장한다
- 등록 성공 시 옷장 목록에 반영된다

출력값:

- 등록된 옷 정보 전체

예외 처리:

- 필수 입력 누락
- 이미지 업로드 실패
- 지원하지 않는 파일 형식 업로드

### 3.2 옷 목록 조회

기능 설명:

사용자는 자신이 등록한 옷 목록을 조회할 수 있다.

상세 조건:

- 로그인한 사용자 본인의 옷만 조회된다
- 카드 형태로 표시한다
- 이미지, 이름, 카테고리, 색상, 핏, 포멀함 점수를 보여준다

출력값:

- 옷 목록 배열

추가 고려:

- 카테고리별 필터
- 핏별 필터
- 포멀함 점수별 필터
- MVP에서는 선택 사항

### 3.3 옷 상세 조회

기능 설명:

사용자는 특정 옷의 상세 정보를 볼 수 있다.

조회 항목:

- 이름
- 이미지
- 카테고리
- 색상
- 핏
- 포멀함 점수
- 소재
- 패턴
- 등록일

### 3.4 옷 삭제

기능 설명:

사용자는 등록한 옷을 삭제할 수 있다.

상세 조건:

- 삭제 전 확인 모달 표시 가능
- 삭제 성공 시 목록에서 즉시 제거된다

예외 처리:

- 존재하지 않는 옷
- 본인 소유가 아닌 옷 삭제 시도

### 3.5 옷 수정

기능 설명:

사용자는 등록한 옷 정보를 수정할 수 있다.

수정 가능 항목:

- 이름
- 카테고리
- 색상
- 핏
- 포멀함 점수
- 소재
- 패턴
- 이미지

비고:

이 기능은 MVP에서 포함해도 되고 후순위로 내려도 된다. 그래도 실사용성을 생각하면 있으면 좋다.

## 4. 이미지 업로드 기능

### 4.1 이미지 업로드

기능 설명:

사용자는 옷 등록 시 이미지를 업로드할 수 있다.

상세 조건:

예외 처리:

- 파일 크기 초과

## 5. 날씨 기능

### 5.1 현재 날씨 조회

기능 설명:

사용자의 위치 또는 선택 지역 기준 현재 날씨를 조회한다.

조회 항목:

- 현재 기온
- 체감 온도
- 날씨 상태
- 강수 여부
- 바람 정보

상세 조건:

- 추천 페이지 진입 시 자동 조회 가능
- 위치 권한 거부 시 기본 지역 설정 또는 직접 지역 선택 방식 사용 가능

출력값:

- 날씨 정보 객체

예외 처리:

- 위치 권한 거부
- 날씨 API 호출 실패
- 지역 정보를 가져오지 못한 경우

### 5.2 추천용 날씨 가공

기능 설명:

AI 추천에 적합한 형태로 날씨 정보를 가공한다.

예시:

- `17도 / 흐림 / 바람 약간 있음`
- `8도 / 맑음 / 추움`
- `24도 / 비 / 습함`

목적:

LLM 프롬프트에 넣기 쉬운 구조로 정리하기 위함

## 6. 일정/상황 프롬프트 기능

### 6.1 프롬프트 입력

기능 설명:

사용자는 오늘의 일정이나 상황을 자유롭게 입력할 수 있다.

입력 예시:

- 나 면접 가
- 오늘 발표 있어
- 친구 만나러 가
- 출근해
- 데이트 가
- 너무 꾸민 느낌은 싫어

상세 조건:

- 텍스트 입력 방식
- 길이 제한 설정 가능
- 공백 입력은 허용하지 않는다

검증 조건:

- 최소 1자 이상
- 최대 글자 수 제한 가능

### 6.2 프롬프트 페이지

기능 설명:

사용자는 별도의 프롬프트 입력 페이지에서 오늘의 상황을 입력하고 추천을 요청할 수 있다.

페이지 구성:

- 현재 날씨 요약
- 사용자 입력창
- 추천 실행 버튼
- 추천 결과 영역

목적:

서비스 핵심 경험을 가장 직관적으로 제공하는 페이지

## 7. AI 추천 기능

### 7.1 코디 추천 요청

기능 설명:

사용자가 추천 요청 버튼을 누르면 시스템은 날씨 정보, 프롬프트, 사용자 옷장 데이터를 기반으로 AI 추천을 생성한다.

입력 데이터:

- 현재 날씨
- 사용자 프롬프트
- 사용자 옷 목록
- 옷 카테고리
- 색상
- 핏
- 포멀함 점수
- 소재
- 패턴

처리 규칙:

- 반드시 등록된 옷 안에서만 추천
- 상황에 맞는 스타일 우선
- 날씨에 맞는 두께와 조합 우선
- 완성된 코디 형태로 출력

출력 데이터:

- 추천 코디 아이템 목록
- 추천 이유
- 스타일 톤 요약

### 7.2 추천 결과 표시

기능 설명:

AI가 생성한 추천 결과를 사용자에게 카드 형태로 보여준다.

표시 항목:

- 추천 조합
- 추천 이유
- 사용된 옷 이미지 또는 이름
- 현재 날씨 요약
- 사용자 입력 프롬프트

예시 출력:

- 추천 코디: 네이비 블레이저 / 화이트 셔츠 / 블랙 슬랙스
- 이유: 17도 내외의 선선한 날씨와 면접 상황에 어울리는 단정하고 포멀한 조합입니다.

### 7.3 추천 실패 처리

기능 설명:

추천 생성에 실패한 경우 사용자에게 오류를 안내한다.

실패 케이스:

- 날씨 데이터를 가져오지 못한 경우
- 사용자 옷장이 비어 있는 경우
- 프롬프트가 비어 있는 경우
- AI API 응답 실패

대응 방식:

- 에러 메시지 노출
- 재시도 버튼 제공 가능

에러 안내 기준:

- 로그인하지 않은 경우 로그인 후 추천을 받을 수 있다고 안내한다.
- 옷장이 비어 있는 경우 먼저 옷을 등록해야 한다고 안내한다.
- 날씨 데이터가 없는 경우 날씨를 불러온 뒤 다시 시도하도록 안내한다.
- AI API 응답 실패 또는 일시적인 과부하가 발생한 경우 잠시 후 재시도하도록 안내한다.

- 날씨 조회 실패
- AI 응답 실패
- 추천 가능한 옷 조합 부족

## 13. MVP 우선순위

필수:

- 회원가입 / 로그인 / 로그아웃
- 인증 상태 유지
- 옷 등록 / 조회 / 삭제
- 이미지 업로드
- 현재 날씨 조회
- 프롬프트 입력
- AI 추천 요청 / 결과 출력

있으면 좋은 것:

- 옷 수정
- 추천 결과 저장
- 히스토리 조회
- 카테고리 필터

후순위:

- 소셜 로그인
- 다중 추천 결과
- 이미지 자동 분석
- 즐겨찾기
- 캘린더 연동

## 14. 기능 흐름 요약

추천 기능 흐름:

1. 사용자가 로그인한다.
2. 옷장에 의류를 등록한다.
3. 추천 페이지에 진입한다.
4. 현재 날씨를 불러온다.
5. 사용자가 오늘의 일정/상황을 입력한다.
6. 시스템이 사용자 옷장 + 날씨 + 프롬프트를 조합한다.
7. AI가 코디를 생성한다.
8. 추천 결과를 화면에 표시한다.
