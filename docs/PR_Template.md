# Pull Request Template

이 프로젝트에서 PR 설명을 작성할 때는 아래 템플릿을 기본 규칙으로 사용한다.

## Template

```markdown
# Pull Request

## What

* 작업 내용 요약

## Why

* 작업 이유

## Changes

* 변경 사항 정리

## How to Test

* 테스트 방법

## Issue

* Closes #

## Checklist

* [ ] 명세서 기준으로 작업했는가
* [ ] 불필요한 코드 제거했는가
* [ ] 기존 기능 영향 없는가
* [ ] 기본 테스트 완료했는가
```

## Writing Rules

- 사용자가 PR 본문 또는 PR 설명을 요청하면 위 템플릿을 기본으로 사용한다.
- 별도 언어 지시가 없으면 PR 내용은 한글 위주로 작성한다.
- 기술 용어, 라이브러리명, API명, 파일명처럼 번역이 부자연스러운 표현만 영어를 유지한다.
- `What`에는 작업 내용을 짧게 요약한다.
- `Why`에는 왜 이 변경이 필요한지 적는다.
- `Changes`에는 실제 변경 사항을 핵심만 정리한다.
- `How to Test`에는 리뷰어가 바로 따라할 수 있는 검증 절차를 적는다.
- `Issue`에는 연결할 이슈가 있으면 `Closes #번호` 형식으로 작성한다.
- `Checklist`는 PR 상태에 맞게 체크하거나 비워 둔다.

## Instruction For Codex

사용자가 PR 요청을 하면 이 문서를 기준으로 PR 제목/본문 또는 PR 설명을 작성한다.
별도 형식을 지정하지 않으면 위 템플릿을 그대로 따른다.
